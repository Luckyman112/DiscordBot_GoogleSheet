# Устав-квиз бот

Discord-бот на [discord.py](https://discordpy.readthedocs.io/), который учит участников сервера уставу через ежедневные вопросы в личных сообщениях. Ответы, прогресс и статус каждого участника хранятся в Google Таблице — отдельная база не нужна.

**Кто может нажать** — везде ниже отмечено тегом:
`[Все]` · `[Модератор]` · `[Авто]` (срабатывает само, не команда)

---

## Как это работает

1. Участник получает роль (`TARGET_ROLE_ID`) → бот сам добавляет его в таблицу и шлёт приветствие в ЛС с объяснением механики.
2. Раз в день (`SEND_TIME`, по UTC) бот выбирает случайный неотвеченный вопрос и присылает его в ЛС.
3. Участник отвечает **одним сообщением прямо в этом чате** — ответ и отметка «отвечено» уходят в таблицу.
4. Если участник не отвечает — на следующий день приходит мягкое напоминание с кнопкой «⏸ Отложить на день». Модераторов подключают только начиная с `ESCALATION_THRESHOLD_DAYS` дня игнора.
5. Когда вопросы в таблице заканчиваются — участник просто перестаёт получать рассылку, пока модератор не добавит новые.

## Справочник команд и кнопок

### Вопросы и прогресс

- **Ежедневный вопрос** `[Авто]` — приходит в ЛС по расписанию. В футере — прогресс («Прогресс: 5/20 отвечено (25%)»). Под вопросом кнопка **❓ Вопрос непонятен** `[Все]` — если формулировка неясна, жалуется модераторам вместо того, чтобы засчитаться как игнор.
- **Напоминание** `[Авто]` — если предыдущий вопрос не отвечен. Кнопка **⏸ Отложить на день** `[Все]` — честный снус: откатывает day-counter на 1, не подключая модераторов раньше времени.
- **`!progress`** `[Все]` — сколько вопросов уже отвечено и сколько всего (считается из таблицы в реальном времени, так что учитывает вопросы, добавленные модераторами только что).
- **`!editanswer <текст>`** `[Все]` — исправить последний отправленный ответ. Работает только в течение `ANSWER_EDIT_WINDOW_SECONDS` (по умолчанию 10 минут) после отправки.

### Для модераторов

- **`!ask @участник`** `[Модератор]` — выдать конкретному человеку случайный неотвеченный вопрос вручную, вне расписания.
- **`!go`** `[Модератор]` — запустить полный цикл рассылки прямо сейчас, не дожидаясь `SEND_TIME`.
- **`!sync`** `[Модератор]` — принудительно синхронизировать таблицу: удалить тех, кто потерял роль, обновить никнеймы, добавить новичков, дорисовать нумерацию/рамки/чекбоксы для новых вопросов.

### Автоматика

- **Регистрация/снятие роли** `[Авто]` — получил `TARGET_ROLE_ID` → добавлен в таблицу с приветствием; потерял роль или вышел с сервера → колонки в таблице удаляются, память очищается.
- **Эскалация модераторам** `[Авто]` — тег `MODERATOR_ROLE_ID` в `ADMIN_CHANNEL_ID`, если участник не отвечает `ESCALATION_THRESHOLD_DAYS`+ дней подряд (и не воспользовался снусом).
- **Канал логов бота** `[Авто]` — технические события и ошибки (не удалось записать в таблицу, у участника закрыты ЛС и т.д.) дублируются в `ADMIN_CHANNEL_ID`.

## Для разработчиков

### Стек

Python 3 · [discord.py](https://discordpy.readthedocs.io/) · Google Sheets (`gspread`, авторизация через `google-auth`) · `python-dotenv`

### Установка

1. **Поставить зависимости:**

   ```bash
   pip install -r requirements.txt
   ```

2. **Создать `.env` в корне проекта** (шаблон — в `.env.example`):

   ```env
   DISCORD_TOKEN=токен_бота
   SHEET_ID=id_гугл_таблицы_из_url
   TARGET_ROLE_ID=id_роли_для_автодобавления
   MODERATOR_ROLE_ID=id_роли_модератора
   ADMIN_CHANNEL_ID=id_канала_для_логов_и_жалоб
   ```

3. **Добавить `credentials.json`** — ключ сервисного аккаунта Google (Google Cloud Console → Service Accounts → Keys → Create key, JSON). Сервисному аккаунту нужен доступ редактора к таблице.

4. **Запустить бота:**

   ```bash
   python main.py
   ```

### Структура таблицы

- Строка 3 — заголовки колонок: `ID-<discord_id>` и `<ник> answ` для каждого участника, парами.
- Строка 2 — чекбокс «Пауза» для каждого участника (`TRUE` — рассылка приостановлена).
- Строка 1 — статусная ячейка (например, «⚠️ ЛС закрыты, свяжитесь!»).
- Колонка B, начиная со строки 4 — автонумерация вопросов.
- Колонка C, начиная со строки 4 — текст вопросов (добавляются модераторами вручную, бот сам подхватывает и дорисовывает чекбоксы/рамки при следующей синхронизации).

### Структура проекта

```
main.py              — точка входа, грузит cogs и стартует бота
config.py             — все настраиваемые ID, время рассылки, инстанс бота, логирование
database.py           — весь слой работы с Google Таблицей и памятью бота (async-обёртка над gspread)
cogs/
  registration.py       — добавление/удаление участников по роли
  reports.py             — рассылка вопросов, приём ответов, прогресс, кнопки, !ask/!progress/!editanswer
  management.py          — запуск бота, !go/!sync, обработка ошибок команд
waiting_answers.json  — память бота (кто чего ждёт), не хранится в git
```

### Проверка изменений

Тестов и линтера в проекте нет — после правок достаточно:

```bash
python -m py_compile <изменённый_файл>.py
```
